## 7 ПРАВИЛ РАЗРАБОТКИ (обязательно для обоих Claude)
1. NO STUBS — только реальный рабочий код
2. ПЛАН ПЕРЕД КОДОМ — подробный план до первой строки
3. МНОГОУРОВНЕВАЯ АРХИТЕКТУРА — структура класса, потом каждая функция
4. ПОДРОБНОЕ ОПИСАНИЕ — перед функцией: цель, вход, выход, логика, ошибки, тест
5. ИТЕРАТИВНАЯ РАЗРАБОТКА — структура→fn1→fn2→интеграция→тест→финиш
6. ТЕСТИРОВАНИЕ ОБЯЗАТЕЛЬНО — локально, реальные данные
7. КАЧЕСТВО > СКОРОСТЬ
