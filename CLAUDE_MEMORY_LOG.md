## 🧠 УРОКИ И ПРАВИЛА ПАМЯТИ

### Урок 1: ПЕРЕПРОВЕРКА
- Я часто спешу и упускаю детали
- **ПРАВИЛО:** Всегда перепроверять ВСЕГДА полный список требований перед финишем
- Не закончил сессию пока не уверен что ВСЁ на месте

### Урок 2: ONE COMMAND = БЕЗ ПРОБЛЕМ
- Когда даю несколько блоков кода подряд - пользователь не может скопировать всё за раз
- **ПРАВИЛО:** ВСЕГДА давать ВСЁ в ОДНОЙ команде (Python скрипт, один cat <<, или один bash скрипт)
- Избегать множественных блоков [block1] [block2] [block3] - это bad UX
- **ПРАВИЛО ДЛЯ ПАМЯТИ:** Если команда сложная - ВСЕГДА использовать Python один liner или один cat << блок

### Урок 3: ДЕТАЛЬНОСТЬ И ПОЛНОТА
- Краткие версии теряют важную информацию
- **ПРАВИЛО:** Документировать ПОЛНОСТЬЮ без сокращений
- Лучше большой файл чем потеря информации
- Всегда можно скролить, но нельзя восстановить потерянное
