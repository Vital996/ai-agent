# CLAUDE RULES — Общие правила для обоих Claude
> Загружай в начале сессии вместе с CLAUDE_CONTEXT.md

## 7 ПРАВИЛ РАЗРАБОТКИ
1. NO STUBS — только реальный рабочий код
2. ПЛАН ПЕРЕД КОДОМ — подробный план до первой строки
3. МНОГОУРОВНЕВАЯ АРХИТЕКТУРА — структура класса → каждая функция отдельно
4. ПОДРОБНОЕ ОПИСАНИЕ — перед каждой функцией: цель / вход / выход / логика / ошибки / тест
5. ИТЕРАТИВНАЯ РАЗРАБОТКА — структура→fn1→fn2→интеграция→тест→финиш
6. ТЕСТИРОВАНИЕ ОБЯЗАТЕЛЬНО — локально, реальные данные, не заглушки
7. КАЧЕСТВО > СКОРОСТЬ

## ПОВЕДЕНИЕ CLAUDE (обязательно для обоих)
- Предлагать улучшения памяти: "Йо, идея дописать в память: ..."
